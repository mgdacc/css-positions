El Flujo del Documento

Antes de romper las reglas, hay que conocerlas. Por defecto, los elementos se apilan uno tras otro (bloques) o uno al lado del otro (en línea).

/* Las coordenadas de posición */
/* Solo funcionan si el elemento NO es estático */

top: 10px;    bottom: 0;
left: 50%;    right: 2rem;
z-index: 10;
2 / 14

1. Position: Static

Es el valor por defecto. El elemento sigue el flujo natural.

div {
  position: static;
  
  /* Estas propiedades son IGNORADAS por completo */
  top: 100px;
  z-index: 99;
}
3 / 14

2. Position: Relative

El elemento permanece en el flujo, pero puedes moverlo relativo a su posición original. Deja un "hueco" donde solía estar.

.caja-movida {
  position: relative;
  
  /* Se mueve 20px hacia abajo y 10px a la derecha 
     desde donde debería estar normalmente. */
  top: 20px;
  left: 10px;
}
4 / 14

3. Position: Absolute

El elemento es removido del flujo normal. Ya no ocupa espacio y "flota" sobre los demás.

.caja-flotante {
  position: absolute;
  
  /* Se posiciona relativo a... */
  top: 0;
  right: 0;
}
5 / 14

¿Absoluto relativo a qué?

Esta es la regla de oro: Un elemento absoluto se posiciona relativo a su ancestro posicionado más cercano.

/* "Ancestro Posicionado" significa cualquier padre
   que tenga position diferente a static. */

.padre { position: relative; } /* EL ANCLA */

.hijo-absoluto { 
  position: absolute; 
  top: 0; /* Top del padre, no de la página */
}
6 / 14

Sin ancestros posicionados...

Si ningún padre tiene posición (relative, absolute, etc.), el elemento absoluto se posiciona relativo al documento inicial (el <body>).

.sin-padre-controlador {
  position: absolute;
  bottom: 0;
  
  /* Se irá al final de toda la página web */
}
7 / 14

Patrón: Padre Relativo / Hijo Absoluto

El patrón de diseño más común para superponer elementos (ej: texto sobre imagen, iconos de notificación).

.tarjeta {
  position: relative; /* Crea la jaula */
}

.etiqueta-precio {
  position: absolute;
  top: 10px;
  right: 10px;
  /* Queda fijo en la esquina de la tarjeta */
}
8 / 14

4. Position: Fixed

Similar a absoluto, pero relativo al Viewport (ventana del navegador). Se queda fijo al hacer scroll.

.navegacion {
  position: fixed;
  top: 0;
  width: 100%;
  
  /* Ideal para menús pegajosos, botones de 
     "Volver arriba" o modales. */
}
9 / 14

5. Position: Sticky

Un híbrido. Es relative hasta que haces scroll y llega a un punto, entonces se comporta como fixed.

.encabezado-tabla {
  position: sticky;
  top: 0; /* Umbral de activación */
  
  /* Se pegará al techo al bajar, pero dejará 
     de pegarse cuando termine su contenedor padre. */
}
10 / 14

Controlando las Capas: Z-Index

Cuando los elementos se superponen, z-index decide cuál va encima. Mayor número = más arriba.

.modal {
  position: fixed;
  z-index: 1000; /* Muy arriba */
}

.contenido {
  position: relative;
  z-index: 1;
}

/* Nota: z-index NO funciona en position: static */
11 / 14

Truco: Centrado Absoluto

Antes de Flexbox, esta era la única forma de centrar algo perfectamente.

.centro {
  position: absolute;
  top: 50%;
  left: 50%;
  
  /* Corrige el desfase del propio tamaño del elemento */
  transform: translate(-50%, -50%);
}

Resumen Comparativo

Valor     | ¿Espacio? | ¿Relativo a?      | ¿Scroll?
------------------------------------------------------
static    | Sí        | Flujo normal      | Se mueve
relative  | Sí (hueco)| Sí mismo          | Se mueve
absolute  | No        | Ancestro pos.     | Se mueve
fixed     | No        | Viewport          | Fijo
sticky    | Sí        | Flujo / Viewport  | Híbrido
13 / 14

Conclusión

El posicionamiento es poder.

// Consejos finales:
1. Usa 'relative' para contenedores y ajustes finos.
2. Usa 'absolute' para superponer cosas dentro de un contenedor.
3. Usa 'fixed' para barras de navegación y popups.
4. No abuses de 'z-index' (evita la guerra del 99999).
